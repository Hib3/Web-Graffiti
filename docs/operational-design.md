# Operational Design

Web Graffiti's purpose is achieved only when the public site can show current public mirror records and make source health visible.

## Success Criteria

- `public/data/records.json` is generated by source adapters, not hand-maintained dummy data.
- `public/data/source-status.json` is generated with per-source success or error status.
- At least one public archive source produces records.
- At least 20 records are generated by default.
- The frontend loads the generated JSON files from GitHub Pages.
- Only `mirrorUrl` is clickable. `hackedUrl` remains display-safe plain text.

## Degraded But Acceptable

Some archive sources may block automation or return challenge pages. A run is still acceptable when at least the minimum viable dataset is produced and failed sources are shown in `source-status.json`.

## Failed

A run must fail and avoid deployment when:

- No source produces records.
- The generated record count is below the minimum threshold.
- Schema validation fails.
- Sorting by `reportedAt` descending fails.
- The source status file is missing or inconsistent.

## Current Reality

OwnzYou and Defacer.Net are confirmed working from the current automation environment. Zone-Xsec and Zone-H adapters are present, but can be unavailable because of anti-bot or JavaScript challenge responses.
